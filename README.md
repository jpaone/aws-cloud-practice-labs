AWS Cloud Practice Labs


This repository documents my hands-on AWS cloud practice labs, with a focus on building, configuring, troubleshooting, and safely deprovisioning cloud infrastructure. The goal of these projects is to demonstrate practical experience with AWS services beyond certification study, including networking, security, access control, documentation, and operational cleanup.

These labs are designed as repeatable technical exercises. Each project includes notes on the services used, the architecture created, configuration steps, issues encountered, and lessons learned. Where applicable, resources were deleted after completion to reduce cost and minimize unnecessary exposure.

Areas Covered

AWS account and environment setup
VPCs, subnets, route tables, and internet gateways
Security groups and network access control
EC2 instance deployment and connectivity
IAM users, roles, permissions, and access policies
Cloud security best practices
Troubleshooting failed configurations
Resource cleanup and cost awareness
Technical documentation for repeatable labs


Purpose
I use this repository to show practical cloud infrastructure experience in a way that is easier to review than certificates alone. The projects here reflect how I approach technical work: build the environment, validate that it works, troubleshoot problems, document the process, and clean up responsibly afterward.
